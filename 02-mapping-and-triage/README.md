# Mapping and Triage

## Purpose

This is the missing layer between recon and vulnerability testing. Do not jump from a URL list directly to payloads. First model how the application works and decide what deserves manual testing.

1. [Application Mapping](application-mapping.md)

6. [Finding Testable Leads](finding-testable-leads.md)

## Outputs

- route and endpoint inventory,
- role and permission model,
- API schema notes,
- high-value workflows,
- prioritized vulnerability leads.
