# Report Template

## Summary

One or two sentences explaining the vulnerability and affected feature.

## Affected Assets

- URL or endpoint:
- Account/role:
- Object/resource:

## Preconditions

- Required account state.
- Required permissions.
- Required target configuration.

## Steps To Reproduce

1. Step one.
2. Step two.
3. Step three.

## Evidence

- Request.
- Response.
- Screenshot if useful.
- OOB interaction if relevant.

## Impact

Explain what an attacker can read, change, execute, or abuse.

## Recommended Remediation

Describe the root-cause fix, not only a symptom-level patch.

## Retest

How to confirm the fix.
