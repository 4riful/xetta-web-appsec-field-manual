# How To Use This Manual

## The Manual Is Phase-Based

The numbered directories are ordered like a real assessment:

1. Orientation
2. Reconnaissance
3. Mapping and triage
4. Vulnerability testing
5. Workflows
6. Toolkit
7. Code review
8. Reporting
9. Reference library

## Do Not Start In Payloads

Payloads are useful after you have a reason to use them. Start from scope, assets, endpoints, roles, and workflows. Then choose the playbook that matches the observed behavior.

## Use Status Labels

- `reviewed`: organized and suitable for normal use.
- `needs-expansion`: source material exists, but the page is not a complete guide.
- `legacy`: preserved for provenance or historical context.

## When Editing

- Improve canonical field-manual docs first.
- Keep raw Notion export unchanged.
- Move link dumps into `08-reference-library/` unless they support a specific guide.
