# Payload Page

## Purpose

Describe the bug class, parser, or sink this payload family targets.

## Use When

- You know the input context.
- You know the expected vulnerable behavior.
- You can validate safely without harming data or users.

## Notes

- Payloads should be grouped by context, not just copied into one list.
- Avoid destructive payloads unless a safe proof is impossible and scope allows it.
- Link back to the bug-class page so readers understand why the payload matters.
