# Tool Note

## Purpose

What job does this tool solve? Name the input it expects and the result it can realistically produce.

## Use When

- The target/scope condition is clear.
- The output can be manually verified.
- The tool fits the workflow better than manual browsing.

## Notes

- A tool result is a lead, not a finding.
- Include rate-limit and scope warnings when automation may be noisy.
- Document false positives and failure modes.
