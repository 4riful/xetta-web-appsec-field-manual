# Report Finding

## Purpose

Turn a reproducible behavior into a clear finding that a triager or engineer can verify.

## Structure

```markdown
# Summary

# Affected Asset

# Preconditions

# Steps To Reproduce

# Evidence

# Impact

# Remediation

# Retest Notes
```

## Fields

- Severity:
- Affected users/roles/tenants:
- Request and response evidence:
- Screenshots or logs:
- Business impact:
- Technical root cause:
- Suggested fix:

## Notes

- Do not overclaim scanner output.
- Show the smallest safe proof of impact.
- Include retest steps so the fix can be verified.
