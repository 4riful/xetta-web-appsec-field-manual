# Bug Class

## Purpose

Explain the bug in practical language: what trust boundary fails, what feature type commonly exposes it, and what impact it can have.

## Testing Shape

1. Identify the feature and boundary.
2. Capture a normal request and response.
3. Change one meaningful control at a time.
4. Confirm behavior from another role/account/session when needed.
5. Save evidence and write the finding clearly.
